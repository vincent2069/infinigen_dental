第二章 相关工作与基本理论

2.X Infinigen室内场景生成基本理论

Infinigen 是一种面向大规模合成数据生成的程序化三维场景生成框架。与传统依赖人工建模、手工摆放和静态资源拼接的方式不同，Infinigen 更强调“语义约束—结构求解—资产实例化”的自动生成范式。对于室内场景而言，其核心目标并不是简单地随机摆放家具，而是在满足空间连通性、功能分区、几何可行性与语义一致性的前提下，自动生成具有合理布局的完整室内环境。`generate_indoors` 即是这一过程在室内场景上的典型实现入口。

从方法上看，室内生成问题可以形式化描述为一个带约束的优化问题。设室内场景状态记为 x，包含房间集合、拓扑关系、门窗位置、物体占位符以及最终资产实例等变量；设约束集合为 C，目标函数为 E(x)，则其一般形式可写为：

min E(x)
subject to c_i(x) = 0 或 c_i(x) <= 0,  c_i ∈ C

其中，目标函数 E(x) 用于度量场景的不合理程度，例如房间面积偏差、空间重叠、通行受阻、家具关系失衡等；约束集合 C 则描述场景必须满足的硬性条件，如厨房必须与餐厅相邻、卫生间面积不低于阈值、家具不能穿模、门需要连通可达区域等。因此，Infinigen 的室内生成本质上是一个多阶段、近似式的约束满足与组合优化过程。

2.X.1 语义驱动的场景建模

`generate_indoors` 的第一步并不是直接生成几何，而是先建立语义层面的场景描述。系统通常将室内环境表示为若干“房间语义”和“物体语义”的组合。例如在住宅场景中，可能包含卧室、厨房、卫生间、客厅等语义标签；在医院或口腔诊疗场景中，则可以扩展为候诊区、导诊区、主走廊、普通诊室和VIP诊室等功能单元。

设房间类型集合为：

R = {r_1, r_2, ..., r_n}

每个房间 r_i 具有面积区间、连通要求、邻接偏好等属性，可表示为：

r_i = (t_i, a_i^min, a_i^max, N_i, S_i)

其中，t_i 表示房间类型，a_i^min 和 a_i^max 分别表示面积下界和上界，N_i 表示邻接关系集合，S_i 表示该房间允许或偏好的物体语义集合。通过这种抽象方式，系统可以先在“功能层”确定空间组织逻辑，再逐步映射到几何布局与具体资产。

2.X.2 房间拓扑图生成

在语义约束确定之后，系统会构建房间之间的拓扑关系图。通常可将室内平面抽象为图 G = (V, E)，其中节点 V 对应房间，边 E 对应房间之间的连通、邻接或开口关系。如果两个房间需要直接相连，则在对应节点之间添加一条边。这样，房间生成问题就从“几何建模”先转化为“图结构求解”问题。

设 A 为房间邻接矩阵，则：

A_ij = 1,  若房间 i 与房间 j 需要直接连通
A_ij = 0,  其他情况

在实际求解中，拓扑图并非完全随机，而是受到功能逻辑驱动。例如：
（1）候诊区通常应连接导诊区；
（2）主走廊应作为若干诊室的公共连接骨架；
（3）VIP诊室除与主走廊相连外，还可能要求更大的独立空间与更高的私密性。

因此，拓扑图生成阶段的任务是：在满足图连通性和语义规则的前提下，为后续平面布局提供一个可行的结构骨架。

2.X.3 平面布局求解

得到拓扑图后，系统需要进一步将抽象节点映射为实际的二维房间区域。这一阶段可视为“从图到平面”的嵌入过程。每个房间被赋予一个矩形或近似矩形区域，系统通过迭代优化使其满足面积约束、相邻关系和整体外轮廓约束。

若第 i 个房间的长和宽分别为 w_i 与 h_i，则其面积为：

A_i = w_i h_i

为了满足给定面积区间，可定义面积惩罚项：

E_area = Σ_i [max(0, a_i^min - A_i) + max(0, A_i - a_i^max)]

若两个房间需要相邻但在当前布局中未接触，则引入邻接惩罚；若两个房间发生重叠，则引入碰撞惩罚。于是总体平面代价函数可写为：

E_layout = λ_1 E_area + λ_2 E_adj + λ_3 E_overlap + λ_4 E_shape

其中，E_shape 用于约束整体建筑外轮廓的规则性。例如在医院场景中，往往希望整体形状尽可能接近矩形，以便体现真实医疗建筑常见的规整布局。若外包矩形面积为 A_bbox，所有房间并集面积为 A_union，则可定义矩形紧致度：

C_rect = A_union / A_bbox

当 C_rect 越接近 1 时，说明布局越接近规则矩形；反之则表示轮廓存在较多凹陷和不规则空洞。在优化过程中，可以通过最小化：

E_shape = 1 - C_rect

来鼓励形成更规整的医院整体布局。

2.X.4 分阶段贪心求解与随机优化

Infinigen 的室内生成并不是一次性联合求解所有变量，而是采用分阶段策略逐步细化。其原因在于：若同时优化房间结构、门窗位置、物体布局和具体资产选择，搜索空间会急剧膨胀，导致求解效率下降。因此，`generate_indoors` 通常遵循以下流程：

（1）根据约束生成房间类型与数量；
（2）建立房间拓扑图；
（3）求解房间平面布局；
（4）在房间内部生成家具或设备占位符；
（5）将占位符替换为实际三维资产；
（6）进行碰撞检查与细节修正。

这种由粗到细的分层求解方式，本质上是一种启发式贪心策略：先确定高层结构，再局部优化低层细节。对于复杂约束冲突，系统还会引入随机搜索或模拟退火等方法进行状态更新。

若当前状态为 x，候选新状态为 x'，其代价增量为：

ΔE = E(x') - E(x)

则在模拟退火框架下，接受新状态的概率为：

P(accept) = 1,              若 ΔE <= 0
P(accept) = exp(-ΔE / T),   若 ΔE > 0

其中 T 为温度参数。该机制允许系统在早期以一定概率接受较差解，从而跳出局部最优；随着温度降低，搜索逐渐收敛到更稳定的布局结果。

2.X.5 占位符机制与资产实例化

在室内物体生成阶段，Infinigen 采用“先占位、后实例化”的思想。也就是说，求解器首先根据语义规则生成抽象占位符，例如“这里应放一张桌子”“该墙边需要一个储物柜”“这个区域应放置候诊长椅”。这些占位符通常带有语义类别、尺寸范围、朝向限制和父子关系信息，但并不立即绑定具体模型。

设占位符集合为：

P = {p_1, p_2, ..., p_m}

每个占位符 p_j 可表示为：

p_j = (s_j, b_j, r_j, o_j)

其中，s_j 为语义标签，b_j 为包围盒尺寸，r_j 为相对空间关系约束，o_j 为朝向或姿态限制。随后，系统通过语义到资产的映射函数，从资产库中选择一个满足该语义的模型完成实例化：

f : Semantics -> AssetSet

例如，若语义为 Bench，则可以映射到候诊长椅资产；若语义为 ReceptionDesk，则可映射到导诊台资产；若语义为 DentalUnit，则可映射到牙椅设备资产。这种解耦设计具有两个重要优势：其一，空间求解与几何资产选择相互独立，提高了整体系统的灵活性；其二，只需扩展语义映射表和约束规则，便可以较低成本迁移到新的场景域，如住宅、办公室、医院和口腔诊所等。

2.X.6 面向医院场景的约束扩展思想

对于医院尤其是口腔诊疗场景，仅依赖住宅模板显然无法得到符合需求的空间结构，因此需要在 `generate_indoors` 框架下进行领域化扩展。其基本思路不是推翻原有生成流程，而是在“房间语义—拓扑关系—家具语义映射”三个层面增加新的约束。

首先，在房间层面，可引入如下结构：一个候诊区、一个导诊区、一条主走廊、N 个普通诊室以及 M 个 VIP 诊室。普通诊室面积约为 25~30 平方米，VIP 诊室面积约为 40~45 平方米，且 VIP 诊室可进一步划分为诊疗区与休息区。若第 k 条走廊连接 n_k 个普通诊室和 m_k 个 VIP 诊室，则可用容量约束近似表示为：

n_k / 8 + m_k / 5 <= 1

该式表示一条走廊最多服务 8 个普通诊室或 5 个 VIP 诊室；当两类诊室混合时，按加权容量共同占用走廊服务能力。

其次，为了使混搭后的整体医院平面尽量呈矩形，除了前述矩形紧致度约束外，还可以对多条走廊长度的均衡性进行优化。设共有 q 条主要走廊，其长度分别为 L_1, L_2, ..., L_q，平均长度为：

L_bar = (1/q) Σ_i L_i

则走廊均衡性惩罚可定义为：

E_corridor = Σ_i (L_i - L_bar)^2

通过最小化 E_corridor，可以避免某一侧走廊过长、另一侧过短的问题，使诊室混搭后的总体轮廓更加对称和紧凑。

最后，在物体层面，需要新增医院专用语义，如牙椅、口腔水槽、医疗柜、治疗台、导诊台、医生椅和候诊长椅等，并将这些语义映射到相应静态资产工厂。这样，场景生成系统才能在“语义正确”的前提下完成医疗家具和设备的自动填充。

2.X.7 `generate_indoors` 的整体流程总结

综合来看，`generate_indoors` 的工作原理可概括为“约束驱动的层次化生成”。其核心流程不是直接合成最终模型，而是按如下顺序逐层推进：

输入语义约束
-> 生成房间集合与拓扑图
-> 求解二维平面布局
-> 生成门、墙、开口等结构元素
-> 在局部空间中放置语义占位符
-> 将占位符映射为具体资产
-> 完成碰撞检查、姿态修正与场景实例化

这一流程体现了程序化建模中的两个重要思想：第一，复杂场景应通过多层抽象逐步构建，而非一次性直接生成；第二，语义约束是保证场景合理性的关键，几何生成只是语义约束的外在表现。从这个角度看，Infinigen 的 `generate_indoors` 不仅是一个工程实现模块，也为复杂室内环境的自动化建模提供了可复用的方法论基础。

2.X.8 本节小结

本节从约束优化、房间拓扑、平面布局、占位符机制和资产实例化等方面，对 Infinigen 中 `generate_indoors` 的基本原理进行了分析。其本质是一个由语义驱动的多阶段优化过程：先建立房间和物体的功能约束，再通过拓扑求解和平面布局得到可行结构，最后结合语义映射完成资产级场景生成。对于医院或口腔诊疗场景而言，该框架具有良好的可扩展性，只需在语义标签、房间约束和资产映射上进行针对性扩展，便可以生成符合医疗场景逻辑的大规模室内环境。
